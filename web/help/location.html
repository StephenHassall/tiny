Reference: Location

import Location from "/lib/tiny/location.js";

Overview

This is used along side the Route class when using a Single Page Application (SPA) system. Because SPA page URLs have the /#!/ part to them, it requires some extra work to get the other URL parts. The Location class is similar to the browser location object.

Usage

All the functions are static getters and therefore do not require you to create a Location object or use (...) parentheses. Just call the function like let pathName = Location.pathname;. 

console.log('href = ' + Location.href);
console.log('protocol = ' + Location.protocol);
console.log('host = ' + Location.host);
console.log('hostname = ' + Location.hostname);
console.log('port = ' + Location.port);
console.log('pathname = ' + Location.pathname);
console.log('search = ' + Location.search);
console.log('hash = ' + Location.hash);

href static

This is the whole URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

You can set the href value to move the page on to another location.

Location.href = 'http://google.com/';

protocol static

The protocol is the very starting part of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

host static

This is the domain address part and the port address section of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

hostname static

This is the domain address part only of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

port static

This is the port address section only of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

pathName static

This is the path part of the URL without any search or hash marks.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

search static

This is the search part of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark

searchParams static

This gets the search part of the URL as an object with parameter value pairs. For example, if the URL is

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark

then calling the Location.searchParams function will return the following.

{ search1: '123', search2: 'hello' }

You will notice that the parameter values are aways text, even when numbers are used.

hash static

This is the hash part of the URL.

http://domain.com:8080/path/page.html?search1=123&search2=hello#hashMark
http://domain.com:8080/path/page.html#!/spaPage1/param1?search1=123&search2=hello#hashMark
